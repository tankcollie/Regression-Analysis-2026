## 1. GradientDescentOLS 实现说明

### 1.1 核心原理

梯度下降通过迭代更新参数来最小化均方误差（MSE）损失函数：

**损失函数**：
$$L(\beta) = \frac{1}{n} \sum_{i=1}^{n} (y_i - X_i\beta)^2$$

**梯度计算**：
$$\frac{\partial L}{\partial \beta} = \frac{2}{n} X^T (X\beta - y)$$

**参数更新**：
$$\beta = \beta - \eta \cdot \nabla L$$

### 1.2 关键特性

| 特性 | 实现方式 | 说明 |
|------|----------|------|
| 学习率 | `learning_rate` | 控制每次更新的步长 |
| 收敛判断 | `tol` | 连续两次 loss 变化小于阈值时停止 |
| 批量策略 | `gd_type` | 支持 `full_batch` 和 `mini_batch` |
| 随机采样 | `batch_fraction` | Mini Batch 时随机抽取指定比例样本 |
| 截距处理 | `fit_intercept` | 自动在 X 前添加全 1 列 |

### 1.3 截距处理方式

本实验采用**内部自动添加截距**的方式：

```python
def _add_intercept(self, X: np.ndarray) -> np.ndarray:
    """添加截距列（全1列）"""
    if not self.fit_intercept:
        return X
    n = X.shape[0]
    return np.column_stack([np.ones(n), X])

```

## 2. Full Batch vs Mini Batch 学习曲线对比

### 2.1 学习曲线图

### 2.2 差异分析

| 特征 | Full Batch | Mini Batch |
|------|-----------|------------|
| **曲线形态** | 平滑、稳定下降 | 有波动、锯齿状 |
| **收敛速度** | 每轮慢（用全部数据） | 每轮快（用部分数据） |
| **计算成本** | 高（O(n) 每轮） | 低（O(batch_size) 每轮） |
| **随机性** | 无（确定性梯度） | 有（随机采样引入噪声） |
| **适用场景** | 小数据集 | 大数据集 |

### 2.3 为什么 Mini Batch 曲线有波动？

Mini Batch 每轮只用部分数据计算梯度，引入的**随机噪声**有助于：
- 跳出局部最优
- 加快收敛速度
- 降低计算成本

### 2.4 为什么两种模式最终收敛值相同？

虽然 Mini Batch 每轮用不同的随机子集，但梯度期望与 Full Batch 相同：

$$E[\nabla L_{\text{mini}}] = \nabla L_{\text{full}}$$

因此，在相同的学习率和迭代次数下，两者收敛到相同的最优解。

**本实验选择 Mini Batch（batch_fraction=0.2），兼顾效率与效果。**

## 3. 学习率调参与最佳学习率

### 3.1 调参结果

| 学习率 | Val R² | Val Adj R² | Val RMSE | 收敛轮数 | 状态 |
|--------|--------|------------|----------|----------|------|
| 0.100000 | 0.9009 | 0.8993 | 72.3167 | 1000 | ✓ **最佳** |
| 0.050000 | 0.9005 | 0.8990 | 72.4372 | 1000 | 正常 |
| 0.010000 | 0.9003 | 0.8988 | 72.5033 | 1000 | 正常 |
| 0.005000 | 0.9005 | 0.8990 | 72.4450 | 1000 | 正常 |
| 0.001000 | 0.5970 | 0.5908 | 145.8038 | 1000 | 欠拟合 |
| 0.000100 | -9.0004 | -9.1535 | 726.2848 | 1000 | 发散 |
| 0.000010 | -13.2044 | -13.4218 | 865.5850 | 1000 | 发散 |

### 3.2 学习率影响分析

| 学习率范围 | 表现 | 原因分析 |
|------------|------|----------|
| **0.1** | 表现最佳 | 步长合适，收敛最快 |
| **0.01-0.05** | 稳定收敛 | 步长适中，稳定下降 |
| **0.001** | 欠拟合 | 步长偏小，1000轮未充分收敛 |
| **≤0.0001** | **发散** | 步长过小，无法在有限迭代内收敛 |

### 3.3 发散原因分析

学习率过小（≤0.0001）时，模型发散的**原因**：

1. **迭代次数不足**：1000 轮迭代不足以让 loss 下降到最优值
2. **梯度消失**：学习率乘上小梯度后，参数更新量极小
3. **数值精度**：loss 变化量小于浮点数精度，但实际尚未收敛

### 3.4 最终选择

**最佳学习率**: `0.1`

选择依据：Val R² 最高（0.9009），收敛速度最快。

## 5. 标准化与防止数据泄露

### 5.1 为什么需要标准化？

梯度下降对特征尺度非常敏感。本实验数据特征尺度差异大：

| 特征 | 范围 | 均值 | 标准差 |
|------|------|------|--------|
| TV_Budget | 0-300 | 138.76 | 69.28 |
| Radio_Budget | 0-150 | 71.20 | 34.53 |
| Social_Budget | 0-220 | 114.69 | 61.41 |

**不标准化的后果**：
- 损失函数等高线呈椭圆形（窄长）
- 梯度下降呈 zig-zag 震荡
- 收敛缓慢甚至不收敛

### 5.2 为什么不能在全部数据上先做标准化？

| ❌ 错误做法 | ✅ 正确做法 |
|------------|------------|
| 先标准化，后划分 | 先划分，后标准化 |
| `scaler.fit_transform(X)` | `scaler.fit_transform(X_train)` |
| 用全部数据计算 μ 和 σ | 只用 Train 集计算 μ 和 σ |

**错误做法的危害**：

| 危害 | 说明 |
|------|------|
| **信息泄露** | Test 集的均值/标准差被用于标准化 Train 集 |
| **结果虚假** | 模型“提前看到”了 Test 集的信息，评估结果过于乐观 |
| **无法部署** | 新数据无法使用 Test 集的统计量进行标准化 |

### 5.3 如何防止数据泄露？

| 步骤 | ✅ 正确做法 | ❌ 错误做法 | 原因 |
|------|------------|------------|------|
| 划分数据 | **先划分，后标准化** | 先标准化，后划分 | Test 集信息会泄露到 Train 集 |
| 计算统计量 | **只用 Train 集** | 用全量数据 | 会引入 Test 集的均值和标准差 |
| 变换 Val/Test | **只 transform** | 重新 fit_transform | 会使用 Val/Test 自己的统计量 |
