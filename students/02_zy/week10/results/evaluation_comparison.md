## 四、为什么 Task 3 是危险的？

Task 3 在交叉验证之前就对全量数据进行了预处理。也就是说，标准化器在计算均值和标准差时，使用了整个数据集的信息，其中既包括训练集，也包括验证集。缺失值填补同样使用了全局均值。

这样做会造成数据泄露，因为验证集的信息提前参与了数据清洗和标准化过程。因此，验证集不再是真正意义上的未知数据。模型在这种情况下得到的评估结果可能会显得更好，但这种好成绩是不可靠的。

## 五、为什么 Task 4 是无泄漏的？

Task 4 将预处理过程放在了每一折交叉验证的循环内部。在每一折中，程序先划分 `X_train` 和 `X_val`，然后只使用 `X_train` 去拟合 `CustomStandardScaler`，得到训练集的均值和标准差。

随后，程序使用训练集学到的均值和标准差去转换 `X_train` 和 `X_val`。整个过程中，验证集 `X_val` 从未参与 `.fit()` 操作，因此避免了数据泄露。

这种流程更接近真实业务场景：模型上线之后，未来数据是不可能提前参与训练阶段的数据清洗和标准化过程的。

## 六、业务解释

从业务角度来看，MAE 和 MAPE 比 R² 更容易解释。

MAE 表示模型预测值与真实值之间的平均绝对误差，可以理解为模型平均会预测错多少销售额。MAPE 表示平均百分比误差，可以理解为模型平均会有百分之多少的预测偏差。

在向业务团队或管理者汇报时，应该优先参考 Task 4 的结果，而不是 Task 3 的结果。虽然 Task 3 的结果可能看起来更好，但它包含了数据泄露，不能代表模型上线后的真实表现。Task 4 的无泄漏结果才更接近真实部署环境下的泛化误差。
## 七、CTO 视角：技术复盘

本项目中的 `CustomStandardScaler` 采用了类似 sklearn Transformer 的接口结构，主要包含三个方法：`fit(X)`、`transform(X)` 和 `fit_transform(X)`。

其中，`fit(X)` 只负责从传入的数据中学习并保存均值和标准差，即 `self.mean_` 和 `self.std_`。同时，它会处理标准差为 0 或缺失的异常情况，避免后续标准化时出现除以 0 的问题。

`transform(X)` 不会重新计算均值和标准差，而是直接使用 `fit()` 阶段已经保存好的 `self.mean_` 和 `self.std_`。如果数据中存在缺失值，也会使用训练阶段学到的均值进行填补。`fit_transform(X)` 则是先执行 `fit(X)`，再执行 `transform(X)`。

在 `good_cross_validation()` 中，数据隔离是在每一折交叉验证循环内部实现的。程序先划分出 `X_train` 和 `X_val`，然后只对 `X_train` 执行：

```python
X_train = scaler.fit_transform(X_train_raw)
```
## 八、AI 辅助学习记录


### 问题 1：什么是数据泄露？为什么在交叉验证前对全量数据做标准化会造成数据泄露？

通过这个问题，我理解到如果在划分训练集和验证集之前，就使用整个数据集计算均值、标准差或缺失值填充值，那么验证集的信息会提前参与训练阶段的数据预处理过程。这样得到的验证结果会偏乐观，不能真实反映模型上线后的泛化能力。

### 问题 2：`CustomStandardScaler` 中的 `fit()`、`transform()` 和 `fit_transform()` 分别应该负责什么？

通过这个问题，我明确了 Transformer 的接口设计。`fit()` 只负责学习并保存均值和标准差；`transform()` 只使用已经保存的参数去处理新数据；`fit_transform()` 则是先拟合再转换。这个结构可以帮助我们在交叉验证中控制哪些数据参与参数学习。

### 问题 3：在 `good_cross_validation()` 中，为什么只能对 `X_train` 使用 `fit_transform()`，而对 `X_val` 只能使用 `transform()`？

这个问题帮助我理解了无泄漏流水线的关键。训练集可以用于学习预处理参数，因此可以执行 `fit_transform()`；验证集必须模拟未来未知数据，不能参与任何参数学习，所以只能使用训练集学到的均值和标准差进行 `transform()`。

### 问题 4：RMSE、MAE 和 MAPE 分别代表什么？为什么业务汇报时 MAE 和 MAPE 更容易解释？

通过这个问题，我理解到 RMSE 会更强调大误差，MAE 表示平均绝对误差，MAPE 表示平均百分比误差。相比 R²，MAE 和 MAPE 更容易向业务团队解释，因为它们可以直接回答“平均预测错多少”和“平均误差百分比是多少”。

### 问题 5：为什么应该向老板汇报 Task 4 的无泄漏结果，而不是 Task 3 中看起来更好的结果？

通过这个问题，我理解到 Task 3 的结果虽然可能更好，但它包含了数据泄露，不能代表真实上线表现。Task 4 的结果虽然可能略差，但它更接近真实业务环境下模型面对未知数据时的表现，因此更适合作为业务决策依据。
